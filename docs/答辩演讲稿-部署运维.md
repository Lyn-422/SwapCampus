部署拓扑

  用户浏览器               服务器 (Windows Server)
      │                       │
      │   HTTPS              │  Nginx (反向代理 + 静态资源)
      ├──────────────────────┤
      │                       ├── Waitress (WSGI) → Django REST API
      │                       ├── Daphne (ASGI) → Channels / WebSocket
      │                       ├── Redis → 消息广播 + 查询缓存
      │                       └── MySQL 8.0 (业务数据)  /  SQLite (开发环境)
      │                       └── MinIO (图片/文件存储)

域名：geleme.online
部署方式：deploy.bat 一键部署脚本（自动安装依赖 → 数据库迁移 → 前端构建 → 启动服务）

关键监控与运维指标
  • API 响应时间：P50 < 100ms, P99 < 500ms（已通过 pytest 基准测试验证）
  • WebSocket 连接稳定性：前端 3s 自动重连 + Channels 心跳保持
  • 图片存储：本地 media/ 目录（开发）/ MinIO S3 对象存储（生产），按年月分目录
  • 文件上传校验：Nginx 20MB 上限 + 服务端 MIME 类型白名单（JPEG/PNG/WebP）
  • 日志：Django 统一异常处理器记录请求耗时与错误栈，按日滚动
  • 数据库备份：Docker Volume 持久化（mysql_data）+ 定期手动导出

可扩展性分析
  • 当前架构可支撑约 500 并发用户（单服务器 Waitress 多线程 + Nginx 缓冲）
  • 扩展路径：Nginx 负载均衡 → 多 Django 实例 → MySQL 读写分离 → Redis 集群 → CDN 加速
  • 100x 用户增长（全北京高校）方案：容器化部署(K8s) + ElasticSearch 全文搜索 + 数据库主从 + 消息队列异步化
