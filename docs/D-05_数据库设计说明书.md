## 1 引言

本文档是 SwapCampus 校园闲置物品交易平台的数据库设计完整说明，涵盖从概念模型到物理模型的全过程。文档目标是：

### 1.2 数据库选型与版本

**数据库**：MySQL 8.0  
**ORM 框架**：Django ORM

**选型理由**：

- **团队熟悉**：所有成员有 MySQL 使用经验
- **Django 原生支持**：Django ORM 对 MySQL 支持完善，迁移工具成熟
- **事务支持**：支持 ACID 事务，满足订单、积分等关键业务需求
- **性能**：MySQL 8.0 性能提升明显，支持窗口函数等高级特性

**环境区分**：
- 开发环境：使用 SQLite（零配置，快速启动）
- 生产/CI 环境：使用 MySQL 8.0

### 3.2 范式说明与反范式取舍

#### 3.2.1 全局达到 3NF

- **1NF**：所有字段原子性，无重复组
- **2NF**：非主属性完全依赖主键，无部分依赖
- **3NF**：无传递依赖，外键关联通过 ID 而非名称

#### 3.2.2 反范式优化

| 表名 | 反范字段 | 理由 | 一致性保障 |
|------|---------|------|-----------|
| orders | buyer_review_count, seller_review_count | 避免频繁查询 review 表统计 | service 层原子更新 |
| products | view_count | 高频更新场景，原子自增 | F('view_count')+1 更新 |
| credit_records | score_after | 方便审计查询，无需回溯计算 | 创建时记录 |

#### 3.2.3 其他设计决策

- **UUID 主键**：使用 UUID 而非自增 ID，避免遍历攻击
- **BaseModel**：所有模型继承 BaseModel，统一 created_at/updated_at
- **软删除**：通过 status 字段实现（如商品 hidden 状态），不物理删除

### 4.2 DDL 脚本

```sql
-- 创建数据库
CREATE DATABASE swapcampus 
    CHARACTER SET utf8mb4 
    COLLATE utf8mb4_unicode_ci;

USE swapcampus;

-- ==================== 用户相关 ====================

-- 用户表
CREATE TABLE users (
    id CHAR(32) PRIMARY KEY,
    username VARCHAR(150) NOT NULL UNIQUE,
    password VARCHAR(128) NOT NULL,
    email VARCHAR(254) DEFAULT '',
    nickname VARCHAR(50) DEFAULT '',
    avatar VARCHAR(100),
    credit_score INT NOT NULL DEFAULT 100,
    campus VARCHAR(30) DEFAULT '',
    bio TEXT,
    is_active BOOL NOT NULL DEFAULT TRUE,
    is_staff BOOL NOT NULL DEFAULT FALSE,
    date_joined DATETIME NOT NULL,
    INDEX idx_credit_score (credit_score),
    INDEX idx_date_joined (date_joined)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 信用记录表
CREATE TABLE credit_records (
    id CHAR(32) PRIMARY KEY,
    user_id CHAR(32) NOT NULL,
    change INT NOT NULL,
    reason VARCHAR(20) NOT NULL,
    description VARCHAR(200) DEFAULT '',
    score_after INT NOT NULL,
    related_order_id CHAR(32),
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (related_order_id) REFERENCES orders(id) ON DELETE SET NULL,
    INDEX idx_user (user_id),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 通知表
CREATE TABLE notifications (
    id CHAR(32) PRIMARY KEY,
    recipient_id CHAR(32) NOT NULL,
    type VARCHAR(20) NOT NULL,
    title VARCHAR(200) NOT NULL,
    content VARCHAR(500) NOT NULL,
    is_read BOOL NOT NULL DEFAULT FALSE,
    related_order_id CHAR(32),
    related_product_id CHAR(32),
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,
    FOREIGN KEY (recipient_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (related_order_id) REFERENCES orders(id) ON DELETE SET NULL,
    FOREIGN KEY (related_product_id) REFERENCES products(id) ON DELETE SET NULL,
    INDEX idx_recipient_read (recipient_id, is_read, created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- ==================== 商品相关 ====================

-- 分类表（支持二级分类）
CREATE TABLE categories (
    id CHAR(32) PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    parent_id CHAR(32),
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,
    FOREIGN KEY (parent_id) REFERENCES categories(id) ON DELETE SET NULL,
    INDEX idx_parent (parent_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 标签表
CREATE TABLE tags (
    id CHAR(32) PRIMARY KEY,
    name VARCHAR(50) NOT NULL UNIQUE,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 商品表
CREATE TABLE products (
    id CHAR(32) PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    description TEXT NOT NULL,
    price DECIMAL(10,2) NOT NULL,
    original_price DECIMAL(10,2),
    condition VARCHAR(20) NOT NULL,
    status VARCHAR(20) NOT NULL DEFAULT 'pending',
    reject_reason VARCHAR(200) DEFAULT '',
    view_count INT NOT NULL DEFAULT 0,
    seller_id CHAR(32) NOT NULL,
    category_id CHAR(32),
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,
    FOREIGN KEY (seller_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (category_id) REFERENCES categories(id) ON DELETE SET NULL,
    INDEX idx_status (status),
    INDEX idx_category (category_id),
    INDEX idx_seller (seller_id),
    INDEX idx_created_at (created_at),
    FULLTEXT idx_search (title, description)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 商品标签关联表
CREATE TABLE product_tags (
    product_id CHAR(32) NOT NULL,
    tag_id CHAR(32) NOT NULL,
    PRIMARY KEY (product_id, tag_id),
    FOREIGN KEY (product_id) REFERENCES products(id) ON DELETE CASCADE,
    FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 商品图片表
CREATE TABLE product_images (
    id CHAR(32) PRIMARY KEY,
    product_id CHAR(32) NOT NULL,
    image VARCHAR(200) NOT NULL,
    order_num INT NOT NULL DEFAULT 0,
    created_at DATETIME NOT NULL,
    FOREIGN KEY (product_id) REFERENCES products(id) ON DELETE CASCADE,
    INDEX idx_product (product_id, order_num)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 收藏表
CREATE TABLE favorites (
    id CHAR(32) PRIMARY KEY,
    user_id CHAR(32) NOT NULL,
    product_id CHAR(32) NOT NULL,
    created_at DATETIME NOT NULL,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (product_id) REFERENCES products(id) ON DELETE CASCADE,
    UNIQUE KEY unique_user_product (user_id, product_id),
    INDEX idx_user (user_id, created_at)

-- 评论表
CREATE TABLE comments (
    id CHAR(32) PRIMARY KEY,

    FOREIGN KEY (author_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (parent_id) REFERENCES comments(id) ON DELETE CASCADE,
    INDEX idx_product (product_id, created_at),

-- 举报表

    handled_at DATETIME,

    updated_at DATETIME NOT NULL,
    FOREIGN KEY (reporter_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (product_id) REFERENCES products(id) ON DELETE CASCADE,
    FOREIGN KEY (handled_by_id) REFERENCES users(id) ON DELETE SET NULL,
    INDEX idx_status (status),
    INDEX idx_product (product_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- ==================== 交易相关 ====================

-- 订单表
CREATE TABLE orders (
    id CHAR(32) PRIMARY KEY,
    buyer_id CHAR(32) NOT NULL,
    seller_id CHAR(32) NOT NULL,
    product_id CHAR(32) NOT NULL,
    status VARCHAR(20) NOT NULL,
    cancel_reason VARCHAR(200) DEFAULT '',
    cancel_by_id CHAR(32),
    completed_at DATETIME,
    buyer_review_count INT NOT NULL DEFAULT 0,
    seller_review_count INT NOT NULL DEFAULT 0,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,
    FOREIGN KEY (buyer_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (seller_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (product_id) REFERENCES products(id) ON DELETE CASCADE,

    INDEX idx_buyer (buyer_id),
    INDEX idx_seller (seller_id),

    INDEX idx_status (status),

    INDEX idx_seller_status (seller_id, status)

    id CHAR(32) PRIMARY KEY,

    created_by_id CHAR(32) NOT NULL,

    confirmed_at DATETIME,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,
    FOREIGN KEY (order_id) REFERENCES orders(id) ON DELETE CASCADE,
    FOREIGN KEY (created_by_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (confirmed_by_id) REFERENCES users(id) ON DELETE SET NULL,

CREATE TABLE reviews (

-- ==================== 聊天相关 ====================

    PRIMARY KEY (conversation_id, user_id),
    FOREIGN KEY (conversation_id) REFERENCES conversations(id) ON DELETE CASCADE,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 消息表
CREATE TABLE messages (
    id CHAR(32) PRIMARY KEY,
    conversation_id CHAR(32) NOT NULL,
    sender_id CHAR(32) NOT NULL,
    content TEXT NOT NULL,
    image VARCHAR(200),
    is_read BOOL NOT NULL DEFAULT FALSE,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,
    FOREIGN KEY (conversation_id) REFERENCES conversations(id) ON DELETE CASCADE,
    FOREIGN KEY (sender_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_conversation (conversation_id, created_at),
    INDEX idx_sender (sender_id),
    INDEX idx_unread (conversation_id, is_read)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

| orders | idx_seller_status | seller_id, status | BTREE | 组合查询 |
| credit_records | idx_user | user_id | BTREE | 用户积分记录 |
| credit_records | idx_created_at | created_at | BTREE | 时间排序 |
| messages | idx_conversation | conversation_id, created_at | BTREE | 会话消息排序 |
| messages | idx_unread | conversation_id, is_read | BTREE | 未读消息查询 |

## 6 安全与合规

### 6.1 权限设计

#### 6.1.1 账号角色划分

| 角色 | 权限范围 | 用途 |
|------|---------|------|
| admin | ALL PRIVILEGES | DBA 数据库管理、备份恢复 |
| app | SELECT, INSERT, UPDATE, DELETE | 应用连接，禁止 DDL |
| readonly | SELECT | 数据分析，后台统计查询 |

#### 6.1.2 最小权限原则

- 应用账号不授予 DROP、CREATE、ALTER 权限
- 敏感操作（删除）通过软删除实现
- 生产环境禁止 root 账号直连

### 6.2 敏感字段处理

#### 6.2.1 加密存储

| 字段 | 算法 | 说明 |
|------|------|------|
| users.password | PBKDF2 + SHA256 | Django 默认，随机盐值，26万次迭代 |

| 场景 | 脱敏方式 | 示例 |
